CATTPSInterfaces Framework: CATITPSSituationFeatures Interface

Usage: an implementation of this interface is supplied and you must use it as is. You should not reimplement it.

interface CATITPSSituationFeatures

This interface is implemented by Datum System features.
Role: Interface to manage Situation Features Modifiers, [PT], [SL] and [PL] relevant for ISO Standard only. Thanks to this interface caller can access to features employed as situation feature (point, line, plane) and set new ones instead if required.

public virtual GetModifierCount( iSearchSFType, const iDRFBoxContext, oNbrOfHit)

Role: Gets modifiers count inside a box of the DRF frame. The returned number is based on the existing group (collection) structuration AND on actually applied situation feature modifier. In anticipation, study of support for a collection of a Situation Feature not addressed by this API; such an investigation requires a modification of the tolerancing set: hence this is not compliant with a "const" qualification of this method.
Caller wishing to apply a Situation Feature onto a collection not being listed by current call is advised to invoke SetModifierDisplay using the list of Datum Features constituting the collection.
Precondition: ISO Standard applied.
Performs an analysis of the specified DRF box to look for a given type of situation features; you can search for all the type of situation features using CATTPSSituationFeatAll enum value.
Parameters:
iSearchSFType
[ in ] Specifies type of the situation feature to search for.
In case all, an inclusive policy is used meaning that as soon as one Datum Feature (or a group) is valid for any of the types, it will increment final count. In consequence, trying to apply a change afterwards may fails in case of non compliance.
iDRFBoxContext
[ in ] First, Second or the Third Box of the DRF to investigate.
oNbrOfHit
[ out ] Number of Datum Features (or groups) supporting situation feature.
This count constitutes the limit is iterative calls Getxx or Setxx provided here along with this interface (legal range goes from 1 to oNbrOfHit).
Returns:
An HRESULT value.
Legal values:
S_OK
Is meaningful of proper execution
S_FALSE
In case no situation feature is applicable
E_FAIL
When iDRFBoxContext is out of range; for instance trying to read Tertiary for reference frame A|B

public virtual SetModifierDisplay( iSFType, const iDRFBoxContext, const iValue, iIndexInBox, const ipiScopeOfDatumFeatures= NULL )

Role: Sets situation feature at given box in DRF frame.
Precondition: ISO Standard applied.
Manages situation feature modifier application into DRF box defined by iDRFBoxContext at specified index.
Pay attention that this step may require further action to assign the convenient geometry (PoinT for [PT], Line for [SL] or plane for [PL]). Activating such an option without providing the proper link to the toplogy is resulting into an invalid Datum System.
Important notice: Pay attention when this method is called (most of the time when turning off a Situation Feature modifier), this can result in collections rework (parenthesis scopes are simplified when possible); it is strongly recommended to run GetModifierCount to get refreshed information about DRF structuration.
Parameters:
iSFType
[ in ] Specifies type of the situation feature.
iDRFBoxContext
[ in ] First, Second or the Third Box of the DRF to investigate.
iValue
[ in ] TRUE is given to set this option active; FALSE has to be provided otherwise.
iIndexInBox
[ in ] Index in between 1 and limit returned by GetModifierCount.
ipiScopeOfDatumFeatures
[ in ] Optional argument: when given it is taken in consideration and offuscate iIndexInBox value.
This argument is useful usualy to specify a collection of Datum Features (content of the list) to be applied the corresponding iModifier situation feature modifier.
Notice: if modifier value, iModifier, tends to remove (M) or (L) display, regular DRF parenthesis simplification process triggered by the end of this call will remove the collection as it does not make sense.
Returns:
An HRESULT value.
Legal values:
S_OK
Is meaningful of proper execution
S_FALSE
Is possibly returned when requested application is already current status when calling this method
E_FAIL
When iDRFBoxContext is out of range; for instance trying to read Tertiary for reference frame A|B
