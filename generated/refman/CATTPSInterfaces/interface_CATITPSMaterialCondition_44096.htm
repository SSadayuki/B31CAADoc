public virtual GetBoundaryClarification( const iDRFBoxContext, oBoundaryClarification, iIndex= NULL )
Gets boundary clarification facet. This call is verifying several conditions to complete properly:
_ first of all, this call is only valid for DRF features created using ASME standard,
_ following check is about material modifier applied on DRF component: to succeed, component at given box in reference frame is applied either (L), least or (M) maximum material condition.
As soon as both above conditions are met, this call allows reading Boundary Value. In case this flag is TRUE, end user can access to the value of the boundary clarification using GetBoundaryValue.
Parameters:
iDRFBoxContext
[ in ] Position in the DRF composition of the box to consider
oBoundaryClarification
[ out ] Content of the attribute Boundary Clarification.
iIndex
[ in ] Optional argument: useful when GetModifierCount is returning a number of datum feature greater than 1 supporting material condition in current box, GetModifierCount.
Returns:
E_FAIL when method is invoked under invalid context: conditions to respect are reminded in method comments, but for short:
_ if ASME or ANSI standard is not current standard,
_ if queried feature is not a DRF entity,
_ if specification of the box range in DRF is either invalid or Datum Feature is supporting neither (L) nor (M) material condition modifier.
otherwise S_OK is meaningful of proper execution.

public virtual GetListOfDatum( const iDRFBoxContext, iIndexInBox, oScopeOfDatumFeatures)
Retrieves the list of datum features in the group corresponding to considered box at given index (implicitly underlying object must be a DRF entity).
This call is valid for ISO as well as for ASME standard.
Parameters:
iDRFBoxContext
[ in ] First, Second or the Third Box of the DRF to investigate.
iIndexInBox
[ in ] Index in between 1 and limit returned by GetModifierCount.
oScopeOfDatumFeatures
[ out ] List of the Datum Features to be impacted by application of material condition modifier. This list is mandatory to be a sub list of the call .
Returns:
S_OK when the value has been correctly retrieved, E_FAIL otherwise.
public virtual GetModifier( oModifier, const iDatum= NULL )
Retrieves Material Condition modifier. Note that second argument is a default one ; it has only to be valuated, when handled object is a DRF instance (Datum Reference Frame terminology in ASME while ISO standard are using Datum System wording) in order to define the Simple Datum in the Reference Frame concerned by the Material Condition.
Parameters:
oModifier
[ out ] Modifier value to set.
iDatum
[ in ] Optional argument; in some simple DRF (only Datum Simple are composing current reference frame), this argument is defining the box in DRF to consider.
Returns:
E_FAIL if iDatum not belongs to Reference Frame; E_FAIL may also suggest to the end user to rather employ another method for DRF. In case DRF is issued of ISO 5459:2011 standard using intricated Datum Features gathering, please invoke method using Refrence Frame box context plus index in box, GetModifierCount.

public virtual GetModifierCount( const iDRFBoxContext, oNbrOfHit)
Runs an analysis of the specified DRF box to see if material condition modifier may apply several times within the considered box.
This call is valid for ISO as well as for ASME standard; underlying object must be a DRF entity.
When ISO Standard is applied, this methods enumerates the number of groups (collection of Datums) for the considered frame box. Total number of hit is then augmented by the number of orphan datums (Datum not embraced within any groups).
Parameters:
iDRFBoxContext
[ in ] First, Second or the Third Box of the DRF to investigate.
oNbrOfHit
[ in ] Number of Datum Features (or groups) supporting material condition modifier. This count constitutes the limit is iterative calls Getxx or Setxx provided here along with this interface (legal range goes from 1 to oNbrOfHit)
Returns:
E_FAIL in case iDRFBoxContext is out of range; for instance trying to read Tertiary for reference frame A|B S_FALSE in case no modifier is applicable S_OK is meaningful of proper execution.

S_FALSE is sent back when setting does not change anything,
otherwise S_OK is meaningful of proper execution.

public virtual SetModifier( const iDRFBoxContext, const iModifier, iIndexInBox, const ipiScopeOfDatumFeatures= NULL )
Sets Material Condition modifier for DRF entity.
ISO Standard important notice: Pay attention when this method is called (most of the time when turning off a Material modifier), this can result in collections rework (parenthesis scopes are simplified when possible); it is strongly recommended to run GetModifierCount to get refreshed information about DRF structuration.
Parameters:
iDRFBoxContext
[ in ] First, Second or the Third Box of the DRF to investigate.
iModifier
[ in ] Modifier value to set.
iIndexInBox
[ in ] Index in between 1 and limit returned by GetModifierCount.
This argument is prefered when ASME Standard is applied; it does also works good whith ISO when invoked just after GetModifierCount.
ipiScopeOfDatumFeatures
[ in ] Optional argument: when given it is taken in consideration and offuscate iIndexInBox value.
This argument is useful in ISO Standard mainly to specify a collection of Datum Features (content of the list) to be applied the corresponding iModifier material condition.
Notice: if modifier value, iModifier, tends to remove (M) or (L) display, regular DRF parenthesis simplification process triggered by the end of this call will remove the collection as it does not make sense.
Returns:
E_FAIL if iDatum not belongs to Reference Frame.
