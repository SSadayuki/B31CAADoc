CATTPSInterfaces Framework: CATITPSOrientationCstOnly Interface

Usage: an implementation of this interface is supplied and you must use it as is. You should not reimplement it.

interface CATITPSOrientationCstOnly

This interface is implemented by Datum System features.
Role: Interface to manage Variable Distance Modifier, "><", relevant for ISO Standard only.
The "><" symbol inside a box within a reference frame definition allows to specify that datum contribute in blocking orientation degrees of freedom only.

Methods


public virtual GetModifier( const iDRFBoxContext, oValue)

Role: Retrieves "><" modifier application into DRF box defined by iDRFBoxContext.
Precondition: ISO Standard applied.
Parameters:
iDRFBoxContext
[ in ] First, Second or the Third Box of the DRF to investigate.
oValue
[ out ] States if "><" option is applied when TRUE value is returned; at the opposite oValue is FALSE.
Returns:
An HRESULT value.
Legal values:
S_OK
The value has been correctly retrieved
E_FAIL
Otherwise
public virtual SetModifier( const iDRFBoxContext, const iValue= TRUE )

Role: Manages "><" modifier application into DRF box defined by iDRFBoxContext.
Precondition: ISO Standard applied.
Parameters:
iDRFBoxContext
[ in ] First, Second or the Third Box of the DRF to investigate.
iValue
[ in ] Optional argument: TRUE (default value) is given to set this option active; FALSE has to be provided otherwise.
Returns:
An HRESULT value.
Legal values:
S_OK
This value is meaningful of proper execution
S_FALSE
This value is possibly returned when requested application is already current status when calling this method; in other words, nothing has been done as "><" modifier was applied as expected
E_FAIL
In case iDRFBoxContext is out of range; for instance trying to read Tertiary for reference frame A|B
