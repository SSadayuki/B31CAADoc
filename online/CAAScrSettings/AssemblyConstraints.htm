AssemblyConstraints Setting Repository
Quick Reference

AssemblyConstraints

Role: Setting to manage Assembly Design (Contraints) options

Detailed role: Assembly Design (Constraints) setting contains all options necessary to Assembly Design(Constraints) commands

The AssemblyConstraints repository has the following characteristics:

  • Primary Tab Page: /Mechanical Design/Assembly Design/Constraints
  • Repository File: Assembly
  • Version: 1
  • Full Compatibility: 1
  • Upward Compatibility: 1

AssemblyConstraints setting repository attributes:

  • CreateWithPublish
    Role: Constraint creation mode
    Detailed role: According to this attribute's value, constraints can be created on any geometry or only on published geometry
    • Type: integer
    • Default value: 0
    • Default Value Role: By default constraint can be created on any geometry
    • Values:
      0 The constraint can be created on any kind of geometry
      1 The constraint can only be created on geometry published on the direct child level
      2 The constraint can only be created on geometry published on any assembly level
  • DetectedMode
    Role: Quick constraint mode
    Detailed role: This attribute is used by commands such as Quick Constraint, which create the most accurate constraint from the context. The priority order may be determined by a predefined list or by the constraints solver
    • Type: integer
    • Default value: 0
    • Default Value Role: By default constraint is created according to the specified order
    • Values:
      0 Use the specified order
      1 Create verified constraint first
  • QuickConstraintOrderedList
    Role: Constraints ordered list for Quick constraints creation
    Detailed role: This attribute is the ordered list, used by commands such as Quick Constraint, and specifies the constraint creation priority. Note that this array of 6 values must not contain any duplicates
    • Type: string
    • Default value: CATAsmPerpendType
  • ReconnectCstOnCCP
    Role: Reconnect constraints on cut/copy paste mode.
    Detailed role: This attribute determines the conditions under which constraints is copied and pasted.
    • Type: integer
    • Default value: 0
    • Default Value Role: By default constraints are never copied and pasted
    • Values:
      0 Constraints are not copied and pasted
      1 Constraints are copied and pasted only after a Copy
      2 Constraints are copied and pasted only after a Cut
      3 Constraints are always copied and pasted
  • RedundantModeSet
    Role: Disable redundancy check mode.
    Detailed role: This attribute is used to control redundancy check when constraint is created
    • Type: integer
    • Default value: 0
    • Default Value Role: By default do redundancy check
    • Values:
      0 Do redundancy check
      1 Disable redundancy check
