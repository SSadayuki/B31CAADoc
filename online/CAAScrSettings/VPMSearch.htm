VPMSearch Setting Repository
Quick Reference

VPMSearch

Role: Setting to handle the VPM Search Properties

Detailed role: Setting to handle the VPM Search Properties

The VPMSearch repository has the following characteristics:

  • Primary Tab Page: /ENOVIA V5 VPM/VPM Search
  • Repository File: VPMSearch
  • Version: 1
  • Full Compatibility: 1
  • Upward Compatibility: 1

VPMSearch setting repository attributes:

  • *
    Role: New VPM NavigatorENOVIA V5 VPM_Product
    • Type: string
    • Default value:
  • CustoDomain
    Role: CustoDomain
    • Type: string
    • Default value: ENOVIA V5 VPM
  • CustoWinType
    Role: CustoWinType
    • Type: string
    • Default value: All Windows
  • IsFetched
    Role: All the search result will be contained in a single page of RESULT window which may not be convenient if the search result in an extremely long list
    • Type: boolean
    • Default value: true
  • IsPreview
    Role: To see the preview of selected object in the RESULT window
    • Type: boolean
    • Default value: true
  • QueryPerPage
    Role: Set the selected rendering engine
    • Type: string
    • Default value:
