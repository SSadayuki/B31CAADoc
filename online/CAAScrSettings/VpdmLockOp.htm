VpdmLockOp Setting Repository
Quick Reference

VpdmLockOp

Role: ENOVIA V5 VPM Options settings

Detailed role: Setting to handle ENOVIA V5 VPM Options at save or at Lock

The VpdmLockOp repository has the following characteristics:

  • Primary Tab Page: /ENOVIA V5 VPM/ENOVIA V5 VPM Options
  • Repository File: VpdmLockOp
  • Version: 1
  • Full Compatibility: 1
  • Upward Compatibility: 1

VpdmLockOp setting repository attributes:

  • AutoLockOp
    Role: Allows automatic unlock on Part Version.
    • Type: boolean
    • Default value: false
  • FetchAssemblyRelation
    Role: Allows to fetch Assembly Relations.
    • Type: boolean
    • Default value: true
  • ForceUnlock
    Role: Force unlock choice.
    • Type: boolean
    • Default value: false
  • IsVPMSession
    Role: VPM Session available.
    • Type: boolean
    • Default value: false
  • LoadCGROption
    Role: Load CGR option available.
    • Type: boolean
    • Default value: false
  • MaxNodeValue
    Role: Allows to choose maximum Number of Nodes which can be processed.
    • Type: integer
    • Default value: 200
  • UIActivatedObj
    Role: Allows to Use the UI activated object.
    • Type: boolean
    • Default value: false
  • VersionPVDR
    Role: Forbid revisioning of Document Revision without revisioning of Part Version.
    • Type: boolean
    • Default value: false
