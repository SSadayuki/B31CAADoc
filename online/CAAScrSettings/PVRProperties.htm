PVRProperties Setting Repository
Quick Reference

PVRProperties

Role: Product View Result settings

Detailed role: Setting to handle Product View Result options at creation or at synchronization.

The PVRProperties repository has the following characteristics:

  • Primary Tab Page: /ENOVIA V5 VPM/Product View Result
  • Repository File: PVRProperties
  • Version: 1
  • Full Compatibility: 1
  • Upward Compatibility: 1

PVRProperties setting repository attributes:

  • ConstraintInPVR
    Role: Allows to support Constraint in PVR.
    • Type: boolean
    • Default value: false
  • FilenameDefinition
    Role: Configure File name definition.
    • Type: string
    • Default value: #PN#_#VN#_#PVR#
  • iSFirstTime
    Role: Use to enable to initialize setting the first time.
    • Type: boolean
    • Default value: true
  • PreventSynchroNestedWrongPVS
    Role: Prevent PVR synchronization if PVS associated is different than the PVS associated with PRC.
    • Type: boolean
    • Default value: false
  • PVRAutoSync
    Role: Enable / Disable default automatic synchronization at PVR creation.
    • Type: boolean
    • Default value: true
  • PVRCreationMode
    Role: Define how many Product View Result CATProduct will be created (Nested PVR concept).
    • Type: integer
    • Default value: 0
    • Values:
      0 One single PVR CATProduct.
      1 Create a PVR CATProduct at each assembly level.
      2 Use ENOVIA V5 component status to know if a nex PVR CATProduct need to be created.
  • PVRSaveMode
    Role: Enable / Disable default automatic synchronization.
    • Type: boolean
    • Default value: true
  • PVRSubAssInPRCctx
    Role: Enable / Disable default option of expand on PRC instead of PV in case of non-filtered PVR.
    • Type: integer
    • Default value: 0
    • Values:
      0 Create PVR with reference model expand.
      2 Create PVR with instance model expand.
  • PVSSaveMode
    Role: Allows to overwrite Filter Capture or save as new.
    • Type: boolean
    • Default value: false
