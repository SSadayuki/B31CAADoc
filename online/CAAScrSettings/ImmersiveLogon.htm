ImmersiveLogon Setting Repository
Quick Reference

ImmersiveLogon

Role: ENOVIA Logon settings

Detailed role: ENOVIA Logon settings

The ImmersiveLogon repository has the following characteristics:

  • Primary Tab Page: /ENOVIA V5 VPM/ENOVIA V5 VPM Logon
  • Repository File: ImmersiveLogon
  • Version: 1
  • Full Compatibility: 1
  • Upward Compatibility: 1
  • Setting controller:
    • Framework_CPP: CATImmENOVIAProvider
    • Framework_IDL: CATImmENOVIAProvider
    • Interface_CPP: CATIImmLogonSettingAtt
    • Interface_IDL: CATIALogonSettingAtt
    • LateType: CATImmLogonSettingCtrl

ImmersiveLogon setting repository attributes:

  • ControlProcess
    Role: Control server process size
    • Type: boolean
    • Default value: false
  • Hosts
    Role: Hosts
    • Type: string
    • Default value:
  • LastSelectedHost
    Role: Last active Hosts
    • Type: string
    • Default value:
  • LastUserName
    Role: Last active user name
    • Type: string
    • Default value:
  • LoadBalancing
    Role: Use the load balancing option. The better server is returned to user
    • Type: boolean
    • Default value: false
  • NbHost
    Role: Number of hosts
    • Type: integer
    • Default value: 0
  • NbRole
    Role: Number of roles
    • Type: integer
    • Default value: 0
  • NbUserRole
    Role: Number of User roles
    • Type: integer
    • Default value: 0
  • Roles
    Role: Roles
    • Type: string
    • Default value:
  • ServerSize
    Role: Server process size in MBytes
    • Type: integer
    • Default value: 500
    • Min value: 100
    • Max value: 20000
    • Step: 1
  • UserRole*
    Role: List of User Roles to keep login history
    • Type: string
    • Default value:
  • WriteInComboHost
    Role: Allow user to add his own host in connection pannel
    • Type: boolean
    • Default value: false
