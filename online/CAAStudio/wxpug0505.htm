Compiling on Unix

This command builds the Unix workspace declared as an image of your current Windows workspace.
Before you begin:
  • The DS Unix bridge must be installed. See Installing C++ Interactive Dashboard.
  • The DS Unix bridge settings must first be set. See Accessing and Customizing Settings.
  • The Windows workspace for which you want to build the Unix image must be opened in the 3DS Workspace Explorer. See Opening a Workspace.
  • The Unix image may possibly be updated with the most recent changes to the Windows workspace. See Updating a Unix Image Workspace.

Before being able to run the written application, and after you have built it, you need to refresh its run time view with the new built objects (exe or dll) and the newly created or modified resources.

  1. On the Tools menu, click Compile on Unix.

    The 3DS - Build Options dialog box opens.

  2. Check Force mkmk update (-u) to force mkmk to rebuild the build time data, even if it already exists.

  3. Check Activate debug mode (-g) to generate debugging data.

  4. Check Activate warning detection (-w) to issue warning messages.

  5. Check Activate edit/continue option (32 bits) to activate the edit and continue debug mode while compiling.

  6. In More options, possibly type additional mkmk options.

  7. Click Build.

    mkmk is run upon the Unix image with the declared options.

The Output windows opens, if it was not already opened, and displays the mkmk log.
