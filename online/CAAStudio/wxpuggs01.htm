Getting Started

Before you begin: Note that:
  • You should have first installed the C++ Interactive Dashboard.
  • You should also have installed a Dassault Systèmes brand, such as CATIA or DELMIA, as well as the associated API CD ROM. The scenario uses some of the installed frameworks to populate the workspace and run the tools.
  • Note that with Windows, the C++ Interactive Dashboard requires the folder C:\temp. Make sure this folder exists on your computer.

Opening C++ Interactive Dashboard

  1. Click Microsoft Visual Studio 2017 shortcut on your desktop, or on the Start menu, point to All Programs, and click Visual Studio 2017.

    Microsoft Visual Studio 2017 opens and the Start Page is displayed.

  2. On the View menu, point to 3DS Views, and click 3DS Workspace Explorer.

    The 3DS Workspace Explorer opens.

  3. Drag the 3DS Workspace Explorer and drop it onto the left hand arrow.

    The 3DS Workspace Explorer is docked on the left of the Microsoft Visual Studio 2017 window.

  4. On the View menu, click Solution Explorer.

  5. Drag the Solution Explorer and drop it onto the right hand arrow.

    The Solution Explorer is docked on the right of the Microsoft Visual Studio 2017 window.

    C++ Interactive Dashboard is ready for creating a workspace.

Creating a Workspace

  1. On the File menu, point to New, and click 3DS Workspace.

    The New Workspace dialog box opens.

  2. In Location, type a folder path to create a new folder, such as E:\MyWorkspace.
    You can also click to select an existing folder and fill the Location box, and then complete the selected path by typing the folder name into which you want to store the workspace frameworks.

    Warning: The folder you set for the workspace must not exist. Otherwise, a prompt is issued to warn you that a workspace cannot be created in an existing, even empty, folder.

  3. In Tools Level, select the tools level you want to use for this workspace, such as V5_6R2021_B31.

  4. Click Finish.

    • The folder E:\MyWorkspace is created if it did not exist.
    • The empty workspace MyWorkspace is displayed in the 3DS Workspace Explorer.
    • An empty project corresponding to this workspace is created and displayed in the Solution Explorer.

Populating the Workspace

  1. In the 3DS Workspace Explorer, right click MyWorkspace, and click Copy Framework.

    The 3DS - Copy Framework dialog box opens.

  2. In Framework names, click .

    The Browse For Folder dialog box opens.

  3. Locate the API CD ROM installation folder (by default C:\Program Files\Dassault Systemes\B31), then in this folder, locate the CAADoc folder, click the CAADialog.edu framework, and click OK.

  4. Click Finish.

    The framework, its modules and data are displayed in the 3DS Workspace Explorer.

  5. Right click the framework, and click Add All to Solution from the contextual menu to display the framework and the modules in the Solution Explorer.

    The projects corresponding to the framework and to the modules are displayed in the Solution Explorer.

Declaring the Workspace Prerequisites

  1. In the 3DS Workspace Explorer, right click the workspace root node MyWorkspace, and click Prerequisites.

    The 3DS - Manage prerequisites dialog box opens.

  2. Click .

    The Select Folder dialog box is displayed.

  3. Locate the prerequisite workspace root folder, that is, the folder where you have installed the brand CD ROM (by default C:\Program Files\Dassault Systemes\B31), click it, and click Select Folder.

    The selected folder is displayed in the 3DS - Manage prerequisites dialog box.

  4. Click OK.

    The workspace prerequisites are updated. The Output window is displayed and shows the output of the mkGetPreq command run.

Building a Module

  1. In the 3DS Workspace Explorer, possibly click the arrow in front of the framework name to expand the CAADialog.edu framework if it is not already expanded..

  2. Right click the module CAADlgOnIdle.m, and click Build with mkmk.

    The 3DS - Build Options dialog box opens.

  3. Click Force mkdata update (-u).

  4. Click Build.

    mkmk starts to build the module. The Output window displays the mkmk build log.

Creating the Run Time View

  1. In the 3DS Workspace Explorer, right click the workspace root node MyWorkspace, and click Refresh Runtime View.

    The run time view is refreshed with the one of the built module. The Output window displays the mkRtv command build log which displays an error for each non built module.

Creating a Test Framework

  1. In the 3DS Workspace Explorer, right click the workspace root node MyWorkspace, and click New Framework.

    The 3DS - New Framework dialog box opens.

  2. In the Framework name box, type CAADialog as the framework name.

  3. Check Test Framework (.tst).

    The tst extension that depicts a test framework is added to the framework name typed above when creating the framework.

  4. Click Finish.

    The framework CAADialog.tst is created in the workspace. Its IdentityCard.xml file is opened in the Code Editor window.

  5. In the 3DS Workspace Explorer, right click CAADialog.tst, and click Refresh Solution.

    The Solution Project is refreshed with CAADialog.tst.

Creating a Test Case

  1. In the Solution Explorer, right click FunctionTests in CAADialog.tst.

  2. In the contextual menu, point to Add, and click New Item.

    The Add New Item dialog box opens.

  3. In Installed, click the triangle in front of 3DS Templates to expand its contents, if not yet expanded.

  4. Click Tests.

    The test case templates are displayed in the central pane.

  5. In the central pane, click XML test case.

  6. In the Name box, type CAADlgOnIdle. This is the name assigned to the test case.

  7. Click Add.

    The CAADlgOnIdle.xml file opens in the Code Editor window.

  8. Edit this file:

    1. In the Target tag, replace the value attribute with This test validates the CAADlgOnIdle module.

    2. In the Command tag, replace the value attribute with CAADlgOnIdle.

    <?xml version="1.0"?>
<!-- COPYRIGHT Dassault Systemes 2019/07/25 -->

<ODT Version="1.0">
  <Definition>
    <Target value="This test validates the CAADlgOnIdle module"/>
  </Definition>
  <Execution>
    <Command value="CAADlgOnIdle"/>
  </Execution>
</ODT>

  9. On the File, menu, click Save CAADlgOnIdle.xml.

    The file is saved in CAADialog.tst\FunctionTests\TestCases.

Running the Test Case

  1. In the 3DS Workspace Explorer, right click the workspace root node MyWorkspace, point to Tests, and click Run Tests.

    The ODT Replay Selection dialog box opens

  2. Click the arrow in front of CAADialog.tst to expand the framework.

  3. Click the arrow in front of TestCases to expand the folder.

     

  4. Check CAADlgOnIdle.xml.

  5. Click Next.

  6. Check Display Traces.

  7. Click Finish.

    mkodt starts replaying the test case. The Output window displays the mkodt log.
